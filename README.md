# Subtly

은근히 새는 구독, 한눈에 — 구독 서비스 관리 모바일 앱

## 프로젝트 개요

Subtly는 넷플릭스, 유튜브 프리미엄, ChatGPT 등 흩어진 구독 서비스를 한 곳에서 관리할 수 있는 앱입니다.
월별 지출 통계, 다가오는 결제 알림, 예산 관리, 월간 리포트 등을 제공합니다.

## 화면 소개

### 1. 로그인 & 온보딩

카카오 계정으로 간편 로그인합니다. 별도 회원가입이 필요 없습니다.
첫 로그인 시 3단계 온보딩이 진행되어 앱의 핵심 가치를 안내합니다.

| 단계 | 내용 |
|------|------|
| 1단계 | "구독을 한눈에" — 흩어진 구독을 한 곳에서 관리 |
| 2단계 | "지출을 파악하세요" — 월별 추이와 카테고리 차트 |
| 3단계 | "결제일을 놓치지 마세요" — 사전 알림으로 불필요한 결제 방지 |

---

### 2. 홈 (대시보드)

앱을 열면 가장 먼저 보이는 화면입니다. 핵심 정보를 한눈에 확인할 수 있습니다.

- **이번 달 구독료** — 총 월간 지출, 연간 예상 금액, 활성 구독 수 (카운트업 애니메이션)
- **예산 진행률** — 설정한 월 예산 대비 현재 지출을 프로그레스 바로 표시. 초과 시 빨간색 경고
- **다가오는 결제** — 가까운 결제일 순으로 최대 5건 표시 (D-day)
- **월간 리포트** — 이번 달 인사이트 카드
  - 총 지출 금액
  - 월 10만원 초과 시 경고
  - 일시정지로 절약한 금액
  - 가장 비싼 구독 알림
  - 약정 종료 30일 이내 알림
- **오프라인 배너** — 서버 연결 없을 때 캐시 데이터로 표시, 노란색 안내 배너

---

### 3. 구독 목록

등록한 전체 구독을 관리하는 화면입니다.

- **검색** — 서비스명으로 실시간 검색
- **카테고리 필터** — 영상, 음악, 게임, 클라우드, 생산성, 뉴스/읽기, 건강/운동, 기타
- **정렬** — 이름순 / 금액순 / 결제일순 칩으로 전환
- **구독 카드** — 서비스 이니셜 아이콘, 이름, 카테고리, 금액, 결제 주기 표시
- **스와이프 삭제** — 카드를 왼쪽으로 밀면 삭제 버튼 (확인 Alert 포함)
- **순차 페이드인** — 카드가 위에서부터 순서대로 등장
- **스켈레톤 로더** — 데이터 로딩 중 카드 형태 플레이스홀더

---

### 4. 구독 추가

새로운 구독을 등록하는 화면입니다. 프리셋과 직접 입력 두 가지 방식을 지원합니다.

**프리셋 (44개)**
- 서비스명 검색으로 빠르게 찾기
- 카테고리별 필터 (전체 / 영상 / 음악 / 생산성 / ...)
- 선택 시 이름, 금액, 주기, 카테고리, 색상, 통화가 자동 입력

**입력 폼**
- **서비스명** — 자동 포커스, Next 키로 다음 필드 이동
- **금액** — 천단위 콤마 자동 포맷, KRW/USD 통화 토글
- **결제 주기** — 매월/매년/매주 (라벨 + 설명 2줄 표시)
- **결제일** — 자주 쓰는 날짜 칩 (1, 5, 10, 15, 20, 25일) + 직접 입력
- **카테고리** — 8개 카테고리 그리드
- **결제 수단** — 신한카드, 국민카드 등 자유 입력
- **구독 시작일/종료일** — 네이티브 날짜 피커 (iOS: 스피너, Android: 캘린더)
- **메모** — 자유 텍스트
- **실시간 검증** — 금액 0 이하, 결제일 1~31 범위 벗어나면 빨간 테두리 + 에러 메시지
- **로딩 상태** — 제출 중 버튼에 스피너 표시, 더블 탭 방지

---

### 5. 구독 상세

개별 구독의 상세 정보를 확인하고 관리합니다.

- **헤더** — 이니셜 아이콘 (서비스 브랜드 컬러 배경), 서비스명, 금액 (통화별 표시), 결제 주기
- **정보 카드** — 카테고리, 결제일, 상태 (활성/일시정지/재개 예정일), 결제 수단, 시작일, 종료일, 등록일, 메모
- **수정** — 수정 화면으로 이동
- **일시정지** — 기간 선택 팝업 (1개월 / 3개월 / 기한 없이)
- **재개** — 일시정지 해제 (pausedUntil 초기화)
- **삭제** — 확인 Alert 후 삭제 (로딩 상태 표시)

---

### 6. 통계

구독 지출 현황을 시각적으로 분석합니다.

- **월별 지출 추이** — 최근 6개월 바 차트
- **카테고리별 비율** — 활성 구독 기준 파이 차트
- **요약 카드** — 월간 지출 / 연간 예상 / 일 평균 / 활성 구독 수
- **지난달 비교** — 전월 금액, 이번달 금액, 변동 금액 + 증감률 (증가 빨강, 감소 초록)
- **비용 TOP 5** — 가장 비싼 구독 순위 (순위 번호 + 서비스명 + 금액)

---

### 7. 설정

앱의 전반적인 설정을 관리합니다.

**알림**
- 결제일 알림 ON/OFF
- 사전 알림 일수 (1~7일)
- 알림 시간 (6~22시)

**테마**
- 시스템 / 라이트 / 다크

**예산**
- 월 구독 예산 금액 입력 → 홈 화면 진행률 바에 반영

**계정**
- 닉네임 표시
- 로그아웃
- 계정 삭제 (전체 데이터 영구 삭제)

**데이터**
- CSV 내보내기 (서비스명, 금액, 주기, 결제일, 카테고리, 상태, 결제수단, 시작일, 종료일, 메모)

**정보**
- 앱 버전
- 개인정보처리방침
- 이용약관

## 인터랙션 & 애니메이션

| 요소 | 설명 |
|------|------|
| 스켈레톤 로더 | 데이터 로딩 시 펄스 애니메이션 카드 플레이스홀더 |
| 카운트업 | 홈 금액이 0에서 실제 값까지 올라가는 애니메이션 |
| 리스트 페이드인 | 구독 카드가 순서대로 60ms 간격으로 등장 |
| 버튼 스케일 | 누를 때 0.96 축소 → 스프링 복귀 |
| 햅틱 피드백 | 버튼 탭 시 Light 진동, 삭제 시 Warning 진동 |
| 스와이프 삭제 | 왼쪽 스와이프 → 빨간 삭제 버튼 (스케일 보간) |
| 탭 배지 | 3일 이내 결제 건수를 홈 탭에 빨간 숫자로 표시 |
| 커스텀 토스트 | 성공: 초록 라운드 카드, 에러: 빨강 라운드 카드 |

## 시작하기

### 사전 요구사항

- Node.js 18+
- JDK 21
- MySQL 8.0+
- 카카오 개발자 앱 (REST API 키)

### 환경 변수 설정

`.env.example`을 참고하여 `.env` 파일을 생성합니다.

```bash
cp .env.example .env
```

### 프론트엔드 실행

```bash
npm install
npm start
```

### 백엔드 실행

```bash
cd backend
./gradlew bootRun
```

### 프리뷰 모드 (백엔드 없이)

`services/api.ts`와 `app/_layout.tsx`의 `PREVIEW_MODE`를 `true`로 변경하면 mock 데이터로 전체 화면을 확인할 수 있습니다.

## API 엔드포인트

### 인증

| Method | Path | 설명 | 인증 |
|--------|------|------|------|
| POST | `/api/auth/kakao` | 카카오 로그인/가입 | 불필요 |
| POST | `/api/auth/refresh` | 토큰 갱신 | 불필요 |
| POST | `/api/auth/logout` | 로그아웃 | 필요 |
| DELETE | `/api/auth/account` | 계정 삭제 | 필요 |

### 구독

| Method | Path | 설명 | 인증 |
|--------|------|------|------|
| GET | `/api/subscriptions` | 구독 목록 조회 | 필요 |
| POST | `/api/subscriptions` | 구독 등록 | 필요 |
| PUT | `/api/subscriptions/{id}` | 구독 수정 | 필요 |
| DELETE | `/api/subscriptions/{id}` | 구독 삭제 | 필요 |
| GET | `/api/subscriptions/summary` | 요약 (월간 지출, 다가오는 결제) | 필요 |

## DB 마이그레이션

| 버전 | 파일 | 설명 |
|------|------|------|
| V1 | `V1__init_schema.sql` | 초기 스키마 (members, refresh_tokens, subscriptions) |
| V2 | `V2__add_subscription_fields.sql` | 일시정지, 시작일, 종료일, 결제수단 필드 추가 |
| V3 | `V3__add_currency_field.sql` | 구독별 통화(currency) 필드 추가 |
